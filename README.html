📚 E-Book Store REST API Spring Boot 3.2 · Java 17 · JPA · H2 In-Memory Database Project Structure ebook-store/ ├── pom.xml └── src/main/ ├── resources/ │ └── application.properties └── java/com/ebookstore/ ├── EbookStoreApplication.java ├── config/ │ └── DataSeeder.java ← seeds 6 sample books on startup ├── entity/ │ ├── Book.java │ ├── User.java │ └── UserBook.java ← join table: user ↔ book ├── repository/ │ ├── BookRepository.java │ ├── UserRepository.java │ └── UserBookRepository.java ├── service/ │ ├── BookService.java │ ├── UserService.java │ └── UserBookService.java ├── controller/ │ ├── BookController.java │ ├── UserController.java │ └── UserBookController.java ├── dto/ │ ├── BookRequest.java │ ├── BookResponse.java │ ├── UserRegistrationRequest.java │ ├── UserResponse.java │ ├── UserBookResponse.java │ └── ApiResponse.java ← generic wrapper {success, message, data} └── exception/ ├── GlobalExceptionHandler.java ├── ResourceNotFoundException.java └── ResourceAlreadyExistsException.java User Endpoints — /api/users POST /api/users/register Register a new user { "name": "Alice Smith", "email": "alice@example.com", "password": "secret123" } GET /api/users Get all registered users GET /api/users/{id} Get user by ID GET /api/users/email/{email} Get user by email DELETE /api/users/{id} Delete user Book Endpoints — /api/books POST /api/books Add a book to the store { "title": "Clean Code", "author": "Robert C. Martin", "description": "A handbook of agile software craftsmanship", "price": 29.99, "genre": "Technology", "available": true } GET /api/books Get all books (optional: ?title=, ?author=, ?genre=) GET /api/books/available Get all available books GET /api/books/{id} Get a book by ID PUT /api/books/{id} Update book details DELETE /api/books/{id} Remove a book User-Book Access Endpoints — /api/users/{userId}/books POST /api/users/{userId}/books/{bookId} User accesses (purchases) a book from the store GET /api/users/{userId}/books Get all books in a user's library DELETE /api/users/{userId}/books/{bookId} Remove a book from user's library Response Format All endpoints return a unified ApiResponse wrapper: { "success": true, "message": "Book accessed successfully", "data": { "id": 1, "userId": 1, "userName": "Alice Smith", "bookId": 3, "bookTitle": "Design Patterns", "bookAuthor": "Gang of Four", "accessedAt": "2024-01-15T10:30:00" } } HTTP Status Codes Status Scenario 200 OK GET / DELETE / PUT success 201 Created POST success (new resource created) 400 Bad Request Validation error or unavailable book 404 Not Found User or Book not found 409 Conflict Duplicate email / user already has book 500 Internal Server Error Unexpected error How to Run Prerequisites Java 17 ✅ is installed. Install Maven if not already: winget install Apache.Maven Build & Start cd ebook-store mvn clean package -DskipTests mvn spring-boot:run Or run the jar: java -jar target/ebook-store-1.0.0.jar H2 Console (Browser) http://localhost:8080/h2-console JDBC URL: jdbc:h2:mem:ebookdb Username: sa Password: (empty) Quick Test with curl # Register a user curl -X POST http://localhost:8080/api/users/register \ -H "Content-Type: application/json" \ -d '{"name":"Alice","email":"alice@test.com","password":"pass123"}' # Browse all books (6 pre-seeded) curl http://localhost:8080/api/books # Search by genre curl "http://localhost:8080/api/books?genre=Technology" # User (id=1) accesses book (id=1) curl -X POST http://localhost:8080/api/users/1/books/1 # View user's library curl http://localhost:8080/api/users/1/books Made with IBM Bob
